# Copilot Studio Agent Using SharePoint-Based Knowledge 🤖

## Lesson Overview

This lesson demonstrates how to build a functional Copilot Studio agent that draws exclusively from designated SharePoint repositories. The agent delivers accurate, evidence-based responses while respecting Microsoft 365 permissions and organizational governance standards.

## Why This Lesson Matters

Large SharePoint repositories often contain authoritative but scattered information that is difficult to search manually. A properly configured Copilot Studio agent solves this by providing scoped, permission-aware access to the exact documents users are authorized to view. The result is faster evidence lookup, reduced manual document review, and improved consistency in client engagements, audits, and internal knowledge work.

## Who This Lesson Is For

- **Cloud engineers and architects** responsible for Microsoft 365 solutions
- **Audit and compliance leads** concerned with data governance
- **Technical consultants and internal delivery teams** supporting client-specific knowledge needs

## Agenda

- **Prerequisites** — Licensing, roles, permissions, and environment setup
- **Chapter 1: Create Your Agent** — Open Copilot Studio, name, describe, and configure your agent
- **Chapter 2: Agent Instructions** — Craft high-quality instructions that enforce accuracy, citation rigor, and failure behavior
- **Chapter 3: Add SharePoint Knowledge** — Connect your agent to a scoped SharePoint library or folder
- **Chapter 4: Authentication & Access Control** — Understand Entra ID auth and permission enforcement
- **Chapter 5: Operational Guidance** — Common questions, advanced capabilities, and scaling considerations
- **Chapter 6: Publish & Best Practices** — Publish securely and apply long-term operational best practices

## Learning Outcomes

By the end of this lesson, participants will be able to:

1. Create a new agent in Copilot Studio with a narrowly scoped SharePoint knowledge source
2. Develop high-quality instructions that enforce accuracy, citation rigor, and appropriate failure behavior
3. Add and validate SharePoint content while preventing context bleed
4. Configure and test the agent while confirming that SharePoint permissions are fully enforced
5. Publish the agent securely and apply operational best practices for long-term reliability
